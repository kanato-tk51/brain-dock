# Prompt Archive: 個人ライフOS設計依頼

- 保存日: 2026-02-20
- 用途: 将来の再実装・再設計時に同じ要件で再実行するため

## Original Prompt

あなたはプロダクト設計・実装に強いエージェントです。以下の「個人ライフOS」を実現するために、(1) どこから着手すべきか（優先順位・段階的ロードマップ）、(2) 各段階の具体的な実装方法（技術選定・構成・データモデル・入出力フロー・安全対策）、(3) 後で実装するときに参照できる TODO/設計メモを GitHub リポジトリに蓄積するための形（Issue/Docs/フォルダ/テンプレ/ラベル設計）まで含めて提案してください。あと、このプロンプトもどこかに保存しておきたい。

前提：
- GitHub にこだわらない。DB をフル活用する可能性が高く、DB が正本（source of truth）になりうる。
- GitHub は主に「設計ドキュメント」「テンプレ」「運用ルール」「TODO（Issue/Projects）」「設計メモ」「実装コード」「エクスポート/監査用スナップショット」の置き場として活用したい。
- 目的は「忘れっぽさを補い、過去の自分の情報を即座に参照し、会話・意思決定・日々の行動を強化する」こと。
- コスト最小（できるだけ無料/低額）で長期運用したい。個人情報・機密は安全に扱う。

# 0. 背景・課題
- 忘れっぽく、会話中に過去の経験ややったことをその場で思い出せず、ベストな回答ができないことがある。
- 自分の経験・学び・意思決定・関係性・行動履歴を、必要なタイミングで“努力なし”に参照できる状態にしたい。
- 会話中に「いま言うべき要点」「過去の関連経験」「ベストな回答案」を提示してくれる会話アシストも将来的に欲しい。

# 1. 管理・蓄積したい対象（個人知識ベース）
## 1-1. 生活・個人
- 日次ログ（体調/気分/睡眠/運動/食事のざっくり）
- ルーティン（朝/夜/週次のチェックリスト）
- 目標（年次/四半期/月次）と振り返り
- 習慣トラッキング（続けたいこと、やめたいこと）
- 価値観メモ（大事にしたいこと、やらないことリスト）
- 気づき・学びの短文メモ（インボックス）
- 好きなものリスト（映画/音楽/本/店/ガジェット/服など）
- 行きたい場所リスト（旅行、店、展覧会、サウナ等）
- 写真や重いメディアは外部リンク管理を基本にしたい

## 1-2. 知識・学習

## 1-4. 人間関係

## 1-5. お金・買い物（扱い注意）

# 2. 重要な制約（必ず守りたい）
- 長期運用・低コスト（無料〜低額）を重視する。
- 個人情報・機密の取り扱いを厳格にする：
  - パスワード、APIキー、住所、電話番号、契約書原本、顧客情報、生データ、健康の詳細などは平文で置かない。
  - 置くなら暗号化 or 外部リンク or 最小限要約にする。
- 継続できる入力フローが最重要：記録負荷は最小、後で整理できる。
- 将来的に「会話アシスト」まで含めてスケールする設計にしたい。
- DB中心でも破綻しないよう、“ポータブルに”設計したい（引っ越し可能性）。

# 3. 将来的に欲しい会話アシスト（リアルタイム）
- 会話中に「いま言うべき要点」「過去の関連経験」「ベストな回答案」を提示。
- 必要要素：
  - 音声入力取得、リアルタイム文字起こし
  - 会話理解（話題/意図/論点抽出）
  - 過去ログ検索（DB中心、必要ならGitHub/ファイルも参照）
  - 提案生成（短く実用的）
  - 安全フィルタ（禁則情報は出さない）
- 音声を保存しない/要約だけ保存などのプライバシー設計も検討。

# 4. 作りたいツール群（最終的には全部作る）
※以下は長期的に全て実装したい機能群。着手順と依存関係を整理し、実装TODOをリポジトリに蓄積できる形で提案してほしい。

## 4-1. 入力（摩擦ゼロ化）
- クイックキャプチャCLI（カテゴリ別追加、1行追記）
- ホットキー入力（Raycast/Alfred/AutoHotkey等）
- スマホ即メモ→自動で日次ログ追記 or PR作成
- ボイスメモ→文字起こし→要約→分類（学び/TODO/人物/決定）
- スクショ/画像→説明文生成→外部リンク保存
- 位置情報ログ→場所ログ更新（住所は保存しない）
- ブラウザ拡張「保存」ボタン（URL+要約）
- （許可範囲のみ）メール/Slack/カレンダー→決定/宿題抽出
- 紙メモ撮影→要点化→日次ログへ
- 雑投入→後で自動整形/分割/タグ付け

## 4-2. 自動整理（勝手に育つ）
- タグ自動付与（topic/people/project/place 等）
- エンティティ抽出→人物/プロジェクト/場所ページへリンク
- 表記ゆれ統合（エイリアス辞書）
- 未整理インボックス掃除（少数だけ分類提案）
- 週次/月次の自動サマリ生成
- 決定ログ抽出、学びログ抽出
- TODO抽出→タスク化
- 締切推定、優先度推定

## 4-3. 検索・参照（人間超え）
- 自然言語検索、断片から候補提示
- 意味検索（類似検索）
- 時間軸ビュー、人物中心ビュー、場所中心ビュー
- “初出”検索（アイデアの初回）
- 閲覧中コンテンツに対する関連ログ提示
- 差分で振り返り（先月の自分と今の自分）

## 4-4. 会話アシスト（最強ゾーン）
- リアルタイム論点抽出
- 過去ログから“言うべきこと候補”提示
- 相手別コミュニケーション最適化
- 禁則事項フィルタ（マスク/警告）
- 会話後の自動記録（要約/決定/宿題）
- 会議中の確認質問提案
- あなたの口調で返答案
- 詰まり検知→論点整理案
- 実績引き出しボタン

## 4-5. 意思決定・思考強化
- 意思決定支援、決定→結果→教訓の追跡
- 迷い検出、価値観整合性チェック
- メタ認知ダッシュボード、未来メモ浮上、リスクレジスター

## 4-6. 目標・習慣・タスクの自動運転
- 今日の3つ提案、週次レビュー自動生成
- 目標進捗推定、習慣の最小単位化
- 買い物意思決定補助、Someday定期浮上

## 4-7. 仕事力ブースト
- 業務チェックリスト自動提示
- 成果の自動ポートフォリオ化
- うまくいった文章のテンプレ化
- 同じミス検知、長文要点抽出
- コード/PR/Issue とログのリンク

## 4-8. 人間関係ブースト
- 次の話題提案、フォローアップ提案
- 紹介グラフ、イベントの気配通知
- 贈り物候補提案（安全配慮）

## 4-9. 旅行・場所ガイド
- 行きたい場所→旅程案、旅行振り返り、再訪提案、表記ゆれ統合

## 4-10. 健康（センシティブ配慮）
- 体調傾向抽出、トリガー候補提示、体調悪い日の最低限モード

## 4-11. 可視化・ダッシュボード
- 今週のハイライト、知識マップ、活動ヒートマップ、人生の目次ポータル

## 4-12. 品質・安全・継続
- 機密スキャナ（電話/住所/鍵っぽい文字列等検出）
- 公開事故防止（誤公開の抑止）
- 暗号化領域ガード（センシティブは暗号化領域へ誘導）
- データ寿命ルール（一定期間後は要約だけ残す等の整理）
- バックアップ/移行ツール（ポータブルにする）

## 4-13. 自分専用エージェント（究極）
- 行動原則エンジン（価値観/やらないことを前提に提案）
- 状況別人格切替（仕事/友達/内省）
- 毎朝ブリーフィング（今日の焦点/注意点）
- 夜のクロージング（3行まとめ/明日の3つ/学び1つ）
- “今の自分”要約カード（直近傾向で自己紹介/強み/関心）
- よく聞かれる質問への回答辞書（経歴/得意/実績/価値観）

## 4-14. 連携・UI
- VS Code拡張（編集/検索/関連リンク）
- Chromeサイドパネル（閲覧内容に関連するメモ表示）
- Slack/Discord 自分用Bot（/memo /recall）
- LINE/Telegram Bot（スマホ最短追記）
- ローカル常駐メニューアプリ（今日の3つ/今週焦点）

## 4-15. 分析（やりすぎ枠、ただし安全配慮）
- 感情/トピック傾向（センシティブ注意）
- 成長曲線推定（何が効いたか）
- 人間関係の偏り可視化（扱い注意）
- 意思決定の癖分析（先延ばし領域等）
- 買い物満足度学習（満足パターン）

# 5. あなたへの依頼（アウトプット要件）
以下を順に、できるだけ具体的に提案してください（実装方法とTODO蓄積方法を重視）。

A) まず何から着手するべきか（最重要）
- 価値が高く、依存関係が少なく、失敗しづらい順に「フェーズ0/1/2/…」としてロードマップ化する
- 各フェーズで「達成できること」「入力方法」「参照できること」「次フェーズへの繋がり」を明示する
- “継続できること”を最優先に、最小の入力で最大のリターンが出る順に並べる

B) 実装の具体案
- データ基盤（DB中心）としての考え方：どういう概念（イベント/人物/場所/プロジェクト/タスク/意思決定/メディアリンク等）が必要か
- 検索・参照の実現方法（全文検索/類似検索/フィルタ/ランキング等）
- 入力の実現方法（CLI/スマホ/ブラウザ/音声等）と、後で整理できる設計
- 会話アシストの実現方法（リアルタイム文字起こし→理解→検索→提案→安全フィルタ→記録）
- 安全設計（機密スキャナ、暗号化、公開事故防止、データ寿命、権限/分離）

C) GitHubリポジトリにTODO/設計メモを溜める設計
- 推奨リポジトリ構成（docs/、design/、notes/、schemas/、scripts/、apps/、exports/ など）
- Issue/Projects の使い分け（機能、バグ、リサーチ、アイデア、改善、セキュリティ等）
- ラベル設計（domain:people/place/task 等、stage:mvp 等、risk:security 等、priority 等）
- PRテンプレ、Issueテンプレ、意思決定記録（ADR）など「後で参照できる」仕組み
- “やりたいこと”を迷子にしないためのバックログ運用案

D) MVP（1〜2週間）で作るべきもの
- 最小で「入力→蓄積→検索→出力」まで一気通貫するスコープを提案
- “忘れっぽさ”の課題に直撃するユースケース（例：会話で実績を引き出す、過去の学びを瞬時に出す）を最優先にする
- そのMVPを基礎として、4.x の全機能へ拡張できるように道筋を提示する

E) 段階的に全機能（4.x）を実現する統合計画
- 依存関係の整理（入力→整理→検索→提示→自動化→会話アシスト→分析…）
- 各段階で追加すべきコンポーネントと、発生しうるリスク/安全配慮
- 最終的に 4.x を包含する全体アーキテクチャの提案

F) 最後に、次の一歩
- 今日から始める具体的TODO（3〜10個）
- リポジトリの初期README案（目的、原則、安全ルール、運用、参照方法）
