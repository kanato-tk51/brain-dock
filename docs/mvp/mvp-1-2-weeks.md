# MVP（1〜2週間）: 入力→蓄積→検索→出力

## MVPゴール
「会話の場で過去の実績・学びを数秒で引き出せる」ことを最短で達成する。

## このMVPで扱うドメイン
- 日記
- TODO
- 学びメモ
- 思考メモ

## スコープ（やること）
1. クイックキャプチャCLI
2. `captures_raw -> notes/tasks` 変換ジョブ（ルールベース）
3. `notes/tasks -> key_facts` 抽出ジョブ
4. FTS検索 + ファクト検索 + フィルタ
5. `recall` コマンド（会話用3行カードを返す）
6. 週次サマリ生成
7. 最低限の機密スキャナ

## スコープ外（後回し）
- 本格リアルタイム音声アシスト
- 高度ダッシュボード
- 複雑な自動優先度推定

## 主要ユースケース
- UC1: 「最近の障害対応実績を1分で話したい」
- UC2: 「同じ失敗を避けるため、過去の反省を即検索したい」
- UC3: 「1on1前に相手との前回論点を確認したい」

## 実装タスク（1〜2週間）
### Day 1-2
- DBスキーマ適用
- CLI `capture` / `search` 雛形
- シークレット検知（regex）

### Day 3-4
- raw -> notes/tasks 変換（簡易分類）
- FTSインデックス更新
- key_facts 抽出（SPO + confidence）
- `recall` コマンド（上位5件 + 3行要約）

### Day 5-7
- タグ付け補助（topic/context）
- 週次サマリバッチ
- エクスポート(JSONL)

### Day 8-10
- スマホWebhook入力
- `conversation prep` 出力（実績/学び/要点）
- README整備、Issue分解、次フェーズ計画

## 完了条件
- 毎日3件以上の入力を無理なく継続できる
- 会話前に30秒以内で関連ログを引ける
- 機密らしき入力がブロックまたはマスクされる

## 4.x拡張への接続点
- `captures_raw` と `notes/tasks` を維持すれば、音声・画像・会話支援を後付けできる
- 将来 `entities` 系テーブルを追加しても `note_links` を介して移行しやすい
